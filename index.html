📄 ESP32 Firmware Snippet — sensor_read.ino 
// AirSense IoT – ESP32 Main Firmware
// Sensors: MH-Z19B (CO2), PMS5003 (PM2.5), MiCS-2714 (NO2)
// Display : 20x4 I2C LCD  |  Cloud: MQTT → ThingSpeak

#include <Wire.h>
#include <LiquidCrystal_I2C.h>
#include <MHZ19.h>
#include <PMS.h>
#include <WiFi.h>
#include <PubSubClient.h>

#define CO2_RX 16
#define CO2_TX 17
#define SDA_PIN 21
#define SCL_PIN 22

LiquidCrystal_I2C lcd(0x27, 20, 4);
MHZ19 mhz19;
HardwareSerial co2Serial(1);
HardwareSerial pmsSerial(2);
PMS pms(pmsSerial);
PMS::DATA pmsData;

WiFiClient espClient;
PubSubClient mqttClient(espClient);

int co2ppm, pm25, no2ppb;

void setup() {
  Wire.begin(SDA_PIN, SCL_PIN);
  lcd.init(); lcd.backlight();
  lcd.print("AirSense IoT v1.0");

  co2Serial.begin(9600, SERIAL_8N1, CO2_RX, CO2_TX);
  mhz19.begin(co2Serial);
  mhz19.autoCalibration(false);

  pmsSerial.begin(9600);
  WiFi.begin(SSID, PASSWORD);
  mqttClient.setServer(MQTT_SERVER, 1883);

  lcd.setCursor(0,1); lcd.print("System Ready!");
}

void loop() {
  // Read CO2
  co2ppm = mhz19.getCO2();

  // Read PM2.5
  if (pms.readUntil(pmsData))
    pm25 = pmsData.PM_AE_UG_2_5;

  // Read NO2 (analog ADC, calibrated)
  int raw = analogRead(NO2_PIN);
  no2ppb = map(raw, 0, 4095, 0, 200);

  // Update LCD
  lcd.setCursor(0,1);
  lcd.printf("CO2: %4d ppm    ", co2ppm);
  lcd.setCursor(0,2);
  lcd.printf("PM2.5:%3d NO2:%3dppb", pm25, no2ppb);
  lcd.setCursor(0,3);
  lcd.printf("AQI:%.0f %s      ", calcAQI(), getStatus());

  // Upload to cloud via MQTT
  if (mqttClient.connected()) {
    String payload = "{\"co2\":" + String(co2ppm) +
                     ",\"pm25\":" + String(pm25) +
                     ",\"no2\":" + String(no2ppb) + "}";
    mqttClient.publish("airsense/data", payload.c_str());
  }

  delay(5000); // 5-second interval
}

AirSense IoT – Project Report

Air Quality & Pollution Level Monitoring Network

CIA 3 | Applied IoT – CSE Batch | February 2026

Project Milestones

✅ M1

Problem Analysis & System Design

  • Identified key pollutants: CO₂, PM2.5, NO₂, O₃, CO
  • Selected ESP32 as core MCU for Wi-Fi + dual UART support
  • Designed multi-node sensor network with 5 deployment zones
  • Defined AQI threshold levels following CPCB/EPA standards
  • Developed system block diagram and GPIO pin mapping
✅ M2

Hardware Development

  • Interfaced MH-Z19B via UART (NDIR CO₂, ±50 ppm accuracy)
  • Integrated PMS5003 laser particulate counter for PM2.5/PM10
  • Connected MiCS-2714 MEMS sensor for NO₂ via ADC with calibration
  • Deployed 20×4 I²C LCD for on-site real-time display
  • Implemented DHT22 for ambient temperature and humidity context
  • Assembled all sensors in weather-proof ABS enclosure
✅ M3

Cloud Integration & Data Analytics

  • Implemented MQTT protocol over TLS for secure data upload
  • Configured ThingSpeak cloud for time-series data storage
  • Built real-time web dashboard with Chart.js visualization
  • Added AQI calculation engine using EPA breakpoint formula
  • Implemented SMS/email alert system for threshold breaches
  • Created public data API endpoint for awareness reporting
✅ M4

Reporting & Preventive Recommendations

  • Collected 72 hours of sensor baseline data
  • Identified peak pollution hours: 7–9 AM and 5–8 PM
  • CO₂ levels correlate with occupancy and ventilation schedules
  • PM2.5 spikes observed during construction activity and diesel exhausts
  • NO₂ elevated near roadside nodes due to vehicular emissions

📋 Key Observations

🫁 CO₂ (Carbon Dioxide)

Range observed: 412–1,850 ppm

Peak hours: Morning rush & occupied classrooms

Threshold crossed: >1000 ppm in indoor zones during classes

Source: Human respiration, poor ventilation, combustion

💨 PM2.5 (Fine Particles)

Range observed: 5–87 μg/m³

Peak events: Diesel vehicle idling, construction dust

Threshold crossed: Moderate to Unhealthy during 6–9 AM

Source: Vehicle exhaust, road dust, construction activities

🏭 NO₂ (Nitrogen Dioxide)

Range observed: 15–112 ppb

Peak hours: Evening traffic congestion (5–8 PM)

Threshold crossed: 53 ppb briefly at roadside nodes

Source: Vehicle combustion, diesel generators

🛡️ Preventive Recommendations

01
Improve Ventilation: Install HVAC systems with CO₂-triggered automatic ventilation in classrooms and offices to prevent CO₂ buildup above 1000 ppm.
02
Traffic Regulation: Implement vehicle-free zones near sensitive areas during peak hours (7–9 AM) to reduce NO₂ and PM2.5 levels.
03
Green Barriers: Plant dense vegetative buffers along roadways. Trees can reduce particulate matter by 20–30% near emission sources.
04
Early Alerts: Deploy the AQI alert system to send real-time SMS notifications to campus administrators when thresholds are breached.
05
Policy Interventions: Share monthly AQI trend reports with local authorities to lobby for stricter emission controls and inspection programs.
06
Expand Sensor Network: Add more sensor nodes to achieve fine-grained spatial pollution mapping and hotspot identification.

⚠️ Challenges Encountered

Sensor Calibration: MiCS-2714 required multi-point calibration with reference gas due to cross-sensitivity to humidity and temperature.
Wi-Fi Drops: Occasional MQTT disconnections resolved by implementing exponential back-off reconnection and local SD-card buffering.
Power Stability: Voltage drops during PMS5003 fan spin-up caused ESP32 resets; resolved with 1000μF decoupling capacitor.
EMI Interference: ADC readings noisy near Servo motor; shielded cables and hardware averaging over 16 samples improved accuracy.
